Declaring variables

Use this guide to help you cultivate good practices when declaring variables.


  1. Use descriptive names: Choose variable names that are descriptive and easy to understand. This makes your code more readable and helps others understand your code.
  2. Use lowercase letters: Variable names in Python should be in lowercase letters, with words separated by underscores (_).
  3. Avoid using built-in names: Avoid using Python built-in function and variable names as your variable names. This can cause conflicts and unexpected behavior.
  4. Use meaningful variable types: It's a good practice to use meaningful variable types. For example, if you're storing a person's age, use an integer variable type instead of a string.
  5. Initialize variables: Always initialize variables with a value, even if it's just None. This helps avoid errors and makes your code more readable.
  6. Use constants: If you have a value that will not change, use a constant. In Python, constants are typically defined in uppercase letters.

Variable scope

In Python 3, variable scope determines where in your code a variable can be accessed and modified. There are four levels of variable scope in Python:


  1. Local scope: A variable that is defined inside a function, has local scope and can only be accessed and modified within that function.
  2. Enclosing scope: A variable that is defined in the outer function of a nested function has enclosing scope and can be accessed and modified by both the outer and inner functions.
  3. Global scope: A variable that is defined outside of any function has global scope and can be accessed and modified from anywhere in the code.
  4. Built-in scope: The built-in scope contains Python's built-in functions and modules, such as print() and math.

It's important to note that if you define a variable with the same name in a function as a variable in a higher scope, the function will use the local variable rather than the variable in the higher scope. This is known as shadowing.

# Global variable
my_global_variable = "Hello, world!"

def my_function():
    # Local variable
    my_local_variable = "Hello, Python!"
    print(my_local_variable)
                    
    # Access global variable
    print(my_global_variable)
                    
    # Change global variable
    global my_global_variable
    my_global_variable = "Hello, everyone!"
                    
# Call the function
my_function()
                
# Access global variable outside of the function
print(my_global_variable)

In this example, my_global_variable is defined in the global scope and can be accessed and modified from anywhere in the code. my_local_variable is defined in the local scope of my_function() and can only be accessed and modified within that function. The global keyword is used to modify my_global_variable from within the function, allowing the value of the variable to be changed globally.

Data types

There are several built-in data types that you can use to store and manipulate different kinds of data. Here are some of the most common data types in Python 3:

  1. Integer (int): Integers are whole numbers that can be positive, negative, or zero. In Python 3, integers have unlimited precision, meaning they can be arbitrarily large or small. #For example:
    42, -7, 0, 1000000000000000000000000.
  2. Float (float): Floats are decimal numbers. They can represent fractional values and can be positive, negative, or zero. #For example:
    3.14, -2.5, 0.0, 1.0.
  3. Boolean (bool): Booleans are either True or False. They are used to represent logical values in Python 3. >>> 2 > 1 #2 is greater than 1
    True
    >>> 3 == 1 #3 is equal to 1
    False
  4. String (str): Strings are sequences of characters enclosed in quotes. They can be single-quoted ('...') or double-quoted ("..."). #For example:
    'Hello, world!', "Python is awesome!".
  5. List (list): Lists are ordered collections of objects. They can contain objects of different types, including other lists. #Lists can contain integers, strings, booleans, and other lists as well. It can also contain elements of different data types.
    [1, 2, 3],
    ['apple', 'banana', 'cherry'],
    [1, 'apple', True].
    [[1, 2, 3], [2, 4, 6], [7, 8]]

    (1, 2, 3),
    ('apple', 'banana', 'cherry'),

    {'name': 'John', 'age': 30, 'city': 'New York'}.

    These are just a few of the data types available in Python 3. Understanding data types is important because it allows you to choose the appropriate data structure for your data and perform operations on it in the most efficient way possible.

Conditional flow

Conditional flow in Python 3 refers to the ability of the language to execute different blocks of code depending on whether certain conditions are met or not. This is typically achieved using if, elif, and else statements.

if condition:
  # Code to execute if the condition is true
			
elif condition:
  # Code to execute if condition is true
  
else:
  # Code to execute if neither condition nor condition2 is true

The if statement is used to test a condition, and if it evaluates to True, the block of code indented below it is executed. If the condition is not True, the elif statement (short for else if) is evaluated, and if its condition is True, its block of code is executed. If neither the if nor the elif conditions are True, the else block of code is executed.

# Set a variable 
x = 10
	
# Test the value of x
if x < o:
  print("x is negative")
elif x == 0:
  print("x is zero")
else:
  print("x is positive")

In this example, we set a variable x to 10 and then use an if statement to test its value. Since x is greater than 0, the if condition is not true, so we move on to the elif statement. Since x is not equal to 0, the elif condition is also not true, so we execute the code in the else block, which prints the message x is positive.


Conditional flow is a fundamental concept in programming and is used extensively in Python 3, as well as in many other programming languages. It allows you to create flexible programs that can adapt to different situations and make decisions based on input or other conditions.

Loops

In Python 3, there are two main types of loops: for loops and while loops.


A for loop is used to iterate over a sequence (such as a list, tuple, or string). The while loop is used to repeatedly execute a block of code as long as a certain condition is met.

# Example of a "for" loop
my_list = [1, 2, 3, 4, 5]

# Iterate over the list and print each element
for element in my_list:
    print(element)

In this example, we define a list called my_list and then use a for loop to iterate over each element of the list. For each element, we print it to the console.

# Example of a "while" loop
# Set a variable
x = 0

# Keep printing a message until x is greater than 5
while x < 5:
    print("x is less than 5")
    x += 1

In this example, we set a variable x to 0 and then use a while loop to repeatedly print the message "x is less than 5" as long as x is less than 5. We also increment x by 1 in each iteration of the loop to ensure that the loop eventually terminates.


Loops are a powerful tool for iterating over collections of data, processing large datasets, and performing repetitive tasks. They allow you to write more efficient and concise code by automating repetitive tasks, and they are an essential part of any programming language.

Logical operators


Logical operators are used to combine two or more conditions to create a more complex condition. Python 3 supports three logical operators: and, or, and not.

  • and : Returns True if both conditions are true. Otherwise, returns False.
  • or : Returns True if at least one of the conditions is true. Otherwise, returns False.
  • not : Returns the opposite of the condition. If the condition is True, returns False, and vice versa.

Logical operators are often used in conditional statements and loops to create more complex expressions.


Here are some examples of logical operators in action:

# Example of the "and" operator
# Define two variables
x = 5
y = 10

# Check if both variables are greater than 0
if x > 0 and y > 0:
    print("Both x and y are greater than 0")

#Example of the "or" operator
# Define two variables
x = 5
y = 10

# Check if at least one variable is greater than 10
if x > 10 or y > 10:
    print("At least one variable is greater than 10")

#Example of the "not" operator
# Define a variable
x = True

# Check if the variable is not True
if not x:
    print("x is not True")
else:
    print("x is True")

In the first example, we use the and operator to check if both x and y are greater than 0. Since they are, the message "Both x and y are greater than 0" is printed to the console.


In the second example, we use the or operator to check if at least one of x or y is greater than 10. Since y is greater than 10, the message "At least one variable is greater than 10" is printed to the console.


In the third example, we use the not operator to check if x is not equal to True. Since x is True, the message "x is True" is printed to the console.


Identity operators


Identity operators in Python 3 are used to compare the memory locations of two objects. They are useful when you need to check whether two objects are the same object or not. Here are the identity operators available in Python 3:


  • is: returns True if the two objects are the same object
  • is not: returns True if the two objects are not the same object

Here are some examples of how to use identity operators in Python 3:


# is operator
x = [1, 2, 3]
y = [1, 2, 3]
z = x
print(x is y)  # False
print(x is z)  # True

# is not operator
x = [1, 2, 3]
y = [1, 2, 3]
z = x
print(x is not y)  # True
print(x is not z)  # False

In the first example, we create two lists, x and y, with the same values. Since they are two different objects with different memory locations, x is y returns False. We also create a new variable z and assign it to x, so x is z returns True.


In the second example, we use the is not operator to check if two objects are not the same object. Since x and y are two different objects, x is not y returns True. Since x and z are the same object, x is not z returns False.


It is important to note that identity operators only compare the memory locations of objects, not their values. To compare the values of objects, you should use the equality operators (== and !=) instead.


Membership operators


Membership operators in Python 3 are used to test whether a value is a member of a sequence or collection. The two membership operators available in Python 3 are:


  • in: evaluates to True if the value is found in the sequence or collection
  • not in: evaluates to True if the value is not found in the sequence or collection

Here are some examples of how to use membership operators in Python 3:


x = 5
list1 = [1, 2, 3, 4, 5]

# Membership operator - in
if x in list1:
    print(f"{x} is in {list1}")
else:
    print(f"{x} is not in {list1}")

# Membership operator - not in
if x not in list1:
    print(f"{x} is not in {list1}")
else:
    print(f"{x} is in {list1}")

The output of the above code will be:

5 is in [1, 2, 3, 4, 5]

Membership operators can be used with other data types as well, such as strings and tuples:


str1 = "Hello, world!"

# Membership operator - in

    print("3 is in tuple1")

The output of the above code will be:


'world' is in str1
3 is in tuple1

Bitwise operators


Bitwise operators in Python 3 are used to perform operations on the binary representations of integers. Here are the bitwise operators available in Python 3:


  • &: bitwise AND
  • |: bitwise OR
  • ^: bitwise XOR (exclusive OR)
  • ~: bitwise NOT
  • <<: bitwise left shift
  • >>: bitwise right shift

Note: In the examples below, we'll use the bin() function to show the binary representation of integers. This function returns a string representing the binary value of the integer argument passed to it.


Here are some examples of how to use bitwise operators in Python 3:


x = 5
y = 3

# Bitwise AND
print(bin(x & y))  # 0b0001

# Bitwise OR
print(bin(x | y))  # 0b0111

# Bitwise XOR
print(bin(x ^ y))  # 0b0110

# Bitwise NOT
print(bin(~x))  # -0b110

# Bitwise left shift
print(bin(x << 1))  # 0b1010

# Bitwise right shift
print(bin(x >> 1))  # 0b0010

Bitwise operators can also be combined with assignment operators to update the value of a variable. For example:


x = 5
x <<= 2  # equivalent to x = x << 2
print(bin(x))  # 0b10100

Bitwise operators can be used to perform various operations such as setting, clearing, or toggling a bit at a specific position. Here are some examples:


# Set the third bit (counting from the right, starting at 0)
x = 5
x |= 1 << 2
print(bin(x))  # 0b101

# Clear the second bit
x = 5
x &= ~(1 << 1)
print(bin(x))  # 0b001

# Toggle the first bit
x = 5
x ^= 1
print(bin(x))  # 0b100

Function declarations

In Python 3, a function is defined using the "def" keyword followed by the function name and parentheses. Any arguments to the function are listed within the parentheses. The code block that makes up the function is indented beneath the function definition line.

def function_name(arg1, arg2, ...):
    # code block
    return result
  • def : This keyword tells Python that you're defining a function.
  • function_name : This is the name of your function. You can name your function anything you want, but it's a good idea to choose a descriptive name that reflects the purpose of the function.
  • (arg1, arg2, ...) : These are the arguments that the function takes. If the function doesn't take any arguments, you can leave the parentheses empty.
  • : This colon tells Python that you're done with the function definition line and that the code block is about to start.
  • # code block : This is where you put the code that your function will execute. This code block can contain any number of statements, including conditional statements, loops, and other function calls.
  • return result: This line specifies what the function should return as its result. If you don't specify a return value, the function will return None by default.

Here's an example function that takes two arguments and returns their sum:

def add_numbers(x, y):
    result = x + y
    return result

Once you've defined a function, you can call it by using its name and passing any required arguments in parentheses:

result = add_numbers(5, 10)
print(result) # Output: 15

This will call the add_numbers function and pass in the arguments 5 and 10. The function will execute its code block and return the sum of the two arguments, which will be stored in the result variable. Finally, the result will be printed to the console.

Object Oriented Programming

Object-oriented programming (OOP) is a programming paradigm that revolves around the concept of objects. In Python 3, you can define classes to create your own custom objects. A class is a blueprint for creating objects, which can have attributes (data) and methods (functions).


class Dog:
    def __init__(self, name, breed, age):
        self.name = name
        self.breed = breed
        self.age = age

    def bark(self):
        print("Woof!")

    def sit(self):
        print(f"{self.name} sits down.")

In this example, we define a class called Dog which has three attributes (name, breed, and age) and two methods (bark and sit). The __init__ method is a special method that gets called when a new object of the class is created, and is used to set the initial values of the object's attributes.


We can create instances (objects) of the Dog class by calling the class with the () operator, passing in the required arguments for the __init__ method:

my_dog = Dog("Rex", "German Shepherd", 3)

In this example, we create a new Dog object called my_dog with the name "Rex", breed "German Shepherd", and age 3.


We can access the object's attributes using dot notation:

print(my_dog.name) # Output: "Rex"
print(my_dog.breed) # Output: "German Shepherd"
print(my_dog.age) # Output: 3

We can also call the object's methods:

my_dog.bark() # Output: "Woof!"
my_dog.sit() # Output: "Rex sits down."

This is just a simple example, but object-oriented programming can be used to create complex systems with many interacting objects. By encapsulating data and functionality within objects, OOP can help make code more modular, easier to maintain, and easier to understand.
